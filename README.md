# reponsivememo

AI 맞춤 기록장 앱 기획안

## 서비스 개요

reponsivememo는 사용자가 기록하고 싶은 목적을 자연어로 입력하면, AI가 그 목적을 분석해 맞춤형 기록장 레이아웃을 생성해주는 범용 기록 앱입니다.

사용자는 표를 직접 설계하거나 템플릿을 고를 필요 없이 "무엇을 기록하고 싶은지"만 입력하면 됩니다. AI는 입력된 목표를 바탕으로 필요한 필드를 제안하고, 정보가 부족하면 추가 질문을 통해 기록 단위, 화면 구성, 분석 기준을 결정합니다.

운동 기록, 공부 기록, 가계부, 감정 일지, 투자 및 트레이딩 기록, 업무 회고, 독서 기록, 이미지 캡처 메모 등 특정 분야에 한정되지 않고, 사용자의 목적에 맞는 기록 시스템을 만들어주는 것을 목표로 합니다.

## 한 줄 정의

> 무엇을 기록하고 싶은지만 말하면, AI가 나만의 기록장을 만들어주는 앱.

## 핵심 컨셉

사용자는 자신의 기록 목적을 자연어로 작성하고, 앱은 그 목적을 반복 가능하고 분석 가능한 기록장 구조로 변환합니다.

예를 들어 "매일 공부한 내용을 기록하고, 복습이 필요한 부분도 관리하고 싶다"라고 입력하면 AI는 과목, 공부 시간, 이해도, 복습 여부, 오답 메모, 복습일 등의 필드를 제안합니다.

"이미지 캡처 후 아이디어를 메모하고 싶다"라고 입력하면 이미지 첨부, 출처 링크, 태그, 메모, 활용 아이디어, 다시 볼 날짜 등의 구조를 제안합니다.

## 플랫폼 구성

### 모바일 앱

일상 속에서 빠르게 기록하고 사진을 첨부하며 간단히 확인하는 데 집중합니다. 운동 직후, 소비 후, 공부를 마친 후, 이동 중 떠오른 생각 등을 빠르게 기록할 수 있습니다.

### PC 웹

넓은 화면에서 기록장을 생성하고 긴 기록을 작성하며, 이미지를 정리하고 검색 및 분석하는 데 적합합니다. 설치 없이 접근할 수 있고 모바일 앱과 동일한 서버 데이터를 사용합니다.

## 핵심 제품 원칙

- 사용자는 기록 양식을 직접 설계하지 않습니다.
- AI는 사용자의 목적을 분석하여 필요한 기록 항목을 제안합니다.
- 정보가 부족하면 AI는 추가 질문으로 세부 사항을 파악합니다.
- 한 번 확정한 기록장 레이아웃은 수정할 수 없습니다.
- 레이아웃을 바꾸고 싶다면 기존 기록장을 복제해 새 버전을 만듭니다.
- 기록은 단순 저장을 넘어 요약, 검색, 분석될 수 있어야 합니다.
- 생성 과정은 유려한 애니메이션으로 시각적 완성도를 갖춰야 합니다.

## 차별점

일반 메모 앱은 사용자가 자유롭게 내용을 적도록 하고, Notion 같은 도구는 사용자가 직접 데이터베이스를 설계해야 합니다.

reponsivememo는 사용자가 템플릿을 고르는 것이 아니라 AI가 목적에 맞춰 템플릿을 생성합니다. 기록 기준이 일관되게 유지되도록 레이아웃을 잠그고, 생성 과정 자체를 시각적으로 보여주는 점이 핵심 차별점입니다.

## 주요 사용자

- 운동 성장 기록을 하는 사람
- 공부량과 복습을 관리하고 싶은 학생
- 소비 패턴을 분석하고 싶은 사람
- 이미지나 자료를 캡처하고 메모를 붙이고 싶은 사람
- 업무 회고나 프로젝트 진행 상황을 기록하고 싶은 전문가
- 감정, 컨디션, 생활 루틴을 기록하고 싶은 사용자
- 투자, 트레이딩, 실험, 취미 등 자신만의 기준으로 데이터를 쌓고 싶은 사람

## 사용자 흐름

1. 첫 접속 시 기록장 목록 화면을 보여주고, 기록장이 없다면 "무엇을 기록하고 싶나요?" 입력 화면으로 안내합니다.
2. 사용자가 자연어로 기록 목적을 입력합니다.
3. AI가 입력을 분석하고 정보가 부족하면 3~5개의 추가 질문을 합니다.
4. AI는 답변을 기반으로 기록장 레이아웃을 생성합니다.
5. 레이아웃은 갑자기 나타나지 않고 조립되는 애니메이션으로 보여집니다.
6. 사용자는 확정 전 레이아웃을 검토하고 필드명, 순서, 선택지 등을 수정할 수 있습니다.
7. 사용자가 레이아웃을 확정하면 해당 구조는 잠기고 이후 수정할 수 없습니다.
8. 사용자는 생성된 기록장에 데이터를 입력하며, 모바일과 PC 웹에서 동일한 구조로 기록을 쌓습니다.

## 기록장 레이아웃 정책

레이아웃은 `작성 중`과 `잠김` 두 상태를 가집니다.

- `작성 중`: 레이아웃을 생성하고 확정하기 전 단계입니다. 필드 추가, 삭제, 이름 변경, 순서 변경이 가능합니다.
- `잠김`: 레이아웃을 확정하면 구조 수정이 불가능합니다. 기록 데이터의 일관성을 유지하기 위한 정책입니다.

개별 기록의 내용은 수정할 수 있지만 필드 구조 자체는 수정할 수 없습니다. 레이아웃을 바꾸고 싶다면 기존 기록장을 복제해 새 버전을 만들어야 합니다.

레이아웃 잠금 안내 문구:

> 이 기록장은 확정 후 레이아웃을 수정할 수 없습니다. 기록 기준의 일관성을 유지하기 위한 설정입니다. 나중에 레이아웃을 바꾸고 싶다면 복제해 새 기록장을 만들 수 있습니다.

## 레이아웃 복제 정책

레이아웃을 확정한 후 수정하고 싶다면 기존 기록장을 직접 편집하는 대신 복제합니다.

복제된 기록장은 `작성 중` 상태로 열리며, 필요한 수정 후 다시 확정하면 잠깁니다. 기존 기록장은 과거 데이터 보존을 위해 그대로 유지됩니다.

## 수익화 정책

기본 서비스는 무료지만 기록장 레이아웃 개수에 제한을 둡니다.

- 무료 사용자는 1개의 기록장만 생성할 수 있습니다.
- 추가 기록장을 만들고 싶다면 기존 기록장을 삭제하거나 유료 플랜으로 업그레이드해야 합니다.
- 기록장을 삭제하면 기존 기록 데이터도 함께 사라질 수 있으므로 삭제 전 데이터 내보내기를 권장합니다.
- 향후 유료 플랜에서는 여러 기록장 생성, 고급 분석, 더 많은 첨부 용량 등을 제공할 수 있습니다.

무료 플랜 제한 안내 문구:

> 무료 플랜에서는 기록장 레이아웃을 1개만 사용할 수 있습니다. 새 기록장을 만들려면 기존 기록장을 삭제하거나 유료 플랜으로 업그레이드하세요.

## 애니메이션 및 인터랙션

애니메이션은 단순 장식이 아니라 이 앱의 핵심 경험입니다. 사용자는 자신의 요청이 구조화되는 과정을 시각적으로 느껴야 합니다.

기본 흐름:

1. 빈 캔버스나 카드가 등장합니다.
2. 기록장을 감싸는 외곽 박스가 부드럽게 나타납니다.
3. 제목과 설명 영역이 자리 잡습니다.
4. 날짜, 텍스트, 숫자, 선택지, 이미지 첨부, 메모 등의 필드가 하나씩 배치됩니다.
5. 필수 입력 항목에 가벼운 강조 효과를 줍니다.
6. 분석에 사용될 대표 지표를 표시합니다.
7. "이 레이아웃으로 확정하기" 버튼이 나타납니다.

이 과정은 2~4초 이내에 진행되어야 하며, 모바일과 PC 웹 모두에서 일관된 느낌을 유지하되 화면 크기에 따라 세부 연출을 조정합니다.

## 화면 구성

### 모바일 앱

모바일 앱은 하단 탭 구조로 단순하게 구성합니다.

- 홈: 최근 기록장과 빠른 기록 버튼
- 기록장: 생성된 기록장 목록
- 검색: 전체 기록 검색
- 내 정보: 계정, 플랜, 데이터 관리, 알림 설정

### PC 웹

PC 웹은 좌측 사이드바를 사용합니다.

- 왼쪽: 기록장 목록
- 중앙: 선택한 기록장의 기록 리스트
- 오른쪽 또는 별도 페이지: 기록 상세와 분석

기록장 생성 화면은 모바일과 PC 모두에서 중요하며, 입력 → 질문 → 생성 → 조립 → 미리보기 → 확정의 단계가 자연스럽게 이어져야 합니다.

## 데이터 구조

| 테이블 | 역할 |
| --- | --- |
| users | 사용자 정보 |
| plans | 플랜 및 구독 정보 |
| record_books | 기록장 기본 정보 |
| record_book_fields | 기록장 필드 구조 |

| attachments | 이미지 및 파일 첨부 |
| tags | 태그 정보 |

| record_book_versions | 기록장 복제 및 버전 관리 |
| record_edit_histories | 기록 수정 이력 |

### 필드 타입

- `text`: 제목, 짧은 메모
- `long_text`: 회고, 일기, 상세 설명
- `number`: 횟수, 점수, 수량
- `money`: 금액
- `date`: 날짜
- `time`: 시간
- `select`: 카테고리, 상태
- `multi_select`: 여러 태그 선택
- `checkbox`: 완료 여부
- `rating`: 만족도, 난이도
- `image`: 사진, 캡처 이미지
- `link`: 참고 URL

## AI 기능 설계

AI는 사용자의 자연어 입력을 분석하여 기록 가능한 구조로 변환합니다.

1. 사용자의 입력을 분석해 기록 목적을 파악합니다.
2. 기록 주기와 단위를 추론합니다.
3. 핵심 입력 필드를 제안하고 정보가 부족하면 추가 질문을 생성합니다.
4. 사용자의 답변을 반영하여 최종 레이아웃을 구성합니다.
5. 필드 타입, 필수 여부, 선택지, 대표 표시 항목을 설정합니다.
6. 분석에 사용할 지표를 제안합니다.

AI가 생성하는 결과는 앱이 자동으로 화면을 렌더링하고 데이터를 분석할 수 있도록 JSON 형식으로 저장합니다.

```json
{
  "recordBookName": "운동 성장 기록장",
  "description": "운동 수행량과 컨디션 변화를 함께 기록하는 기록장",
  "recordUnit": "daily",
  "fields": [
    { "name": "운동 날짜", "type": "date", "required": true },
    {
      "name": "운동 부위",
      "type": "multi_select",
      "options": ["가슴", "등", "하체", "어깨", "팔", "전신"],
      "required": true
    },
    { "name": "주요 운동", "type": "text", "required": true },
    { "name": "총 운동 시간", "type": "number", "unit": "분", "required": false },
    { "name": "오늘의 컨디션", "type": "rating", "required": false },
    { "name": "메모", "type": "long_text", "required": false }
  ],
  "summaryMetrics": ["운동 횟수", "운동 시간 변화", "부위별 빈도", "컨디션 평균"]
}
```

필드 수는 기본적으로 5~10개 사이로 제한하여 기록 피로도를 낮춥니다.

## 기술 스택 제안

| 영역 | 추천 기술 |
| --- | --- |
| 모바일 앱 | React Native + Expo |
| PC 웹 | Next.js |
| 백엔드 | Supabase 또는 NestJS/FastAPI |
| DB | PostgreSQL |
| 파일 저장 | Supabase Storage 또는 S3 호환 스토리지 |
| 인증 | Supabase Auth, Firebase Auth 또는 자체 JWT |
| AI | OpenAI API 또는 유사 LLM API |
| 애니메이션 | Framer Motion, React Native Reanimated |
| 결제 | Stripe, Toss Payments 등 |
| 배포 | Vercel, 클라우드 서버 |
| 제품 분석 | PostHog, Amplitude 또는 자체 로그 |

초기에는 Supabase + Next.js + React Native/Expo 조합이 빠른 MVP 제작에 유리합니다. 이후 필요에 따라 백엔드를 분리하거나 확장할 수 있습니다.

## MVP 개발 범위

1. 사용자가 기록 목적을 입력합니다.
2. AI가 필요한 추가 질문을 합니다.
3. AI가 맞춤 기록장 레이아웃을 생성합니다.
4. 레이아웃이 애니메이션으로 조립되어 보입니다.
5. 사용자가 미리보기에서 레이아웃을 검토하고 확정합니다.
6. 확정된 레이아웃은 잠기며 수정할 수 없습니다.
7. 사용자는 해당 기록장에 데이터를 입력합니다.
8. 무료 사용자는 기록장을 1개만 생성할 수 있습니다.
9. 두 번째 기록장을 생성하려 하면 삭제 또는 유료 전환 안내가 나타납니다.

## 주요 리스크와 대응

- 범용성으로 인한 정체성 부족: "무엇이든 기록하세요"보다 "목적에 맞는 기록장을 만들어줍니다"라는 메시지를 강조합니다.
- AI 레이아웃 품질 문제: 정보가 부족할 때 질문을 충분히 하고, 생성 필드 수를 제한해 복잡성을 줄입니다.
- 레이아웃 수정 불가에 대한 불만: 확정 전 미리보기와 수정 기회를 제공하고, 복제 기능을 안내합니다.
- 무료 플랜 제한으로 인한 이탈: 무료 사용자가 1개의 기록장으로도 충분한 가치를 느끼도록 설계합니다.
- 애니메이션 과도화: 핵심 생성 순간에는 부드러운 연출을 제공하고, 반복 사용 영역에서는 빠른 애니메이션을 적용합니다.
- 기록 작성 피로도: 필드 수를 제한하고 필수 입력을 최소화합니다.

## 브랜딩 방향

이 앱은 단순 메모장처럼 가볍게 보이기보다, 사용자의 삶과 업무를 구조화해주는 지능형 도구로 포지셔닝합니다.

브랜드 톤은 차갑고 복잡한 생산성 도구보다 부드럽고 정제된 개인 기록 시스템에 가깝게 잡습니다.

예시 문구:

> 무엇을 기록하고 싶은지만 말하세요. 나만의 기록장이 만들어집니다.

> 목적을 말하면, 기록 구조가 완성됩니다.

> 기록하고 싶은 것을 입력하면, AI가 나만의 기록장을 설계합니다.

## 최종 정리

reponsivememo는 모바일 앱과 PC 웹으로 제공되는 AI 맞춤 기록장 앱입니다. 사용자는 기록하고 싶은 목적을 자연어로 입력하기만 하면 됩니다.

AI는 추가 질문을 통해 목적을 정확히 파악하고 맞춤형 기록장 구조를 생성합니다. 생성 과정은 유려한 애니메이션으로 표현되어 사용자가 자신의 요청이 구조화되는 과정을 시각적으로 확인할 수 있습니다.

한 번 확정한 레이아웃은 수정할 수 없으며, 기록 기준의 일관성을 유지하기 위해 복제 기능을 제공합니다. 무료 사용자는 기록장 하나만 만들 수 있으며, 여러 기록장을 운영하려면 유료로 전환해야 합니다.

이 앱의 핵심 목표는 사용자가 목적에 맞는 기록을 쉽게 시작하고, 일관된 데이터로 자기 삶과 업무를 분석하게 만드는 것입니다.
